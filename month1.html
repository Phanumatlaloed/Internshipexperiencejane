อ่านเรื่องราว ดูภาพผลงาน
Key mindset: ระบบเก่าอัปเดตเสี่ยง → ต้อง “วางแผนก่อนแก้จริง” + “ทดสอบซ้ำ”

Quick Summary

  • ทำหน้า About Us บน WordPress + WPBakery (เวอร์ชันเก่า)
  • ปรับ Layout ด้วย Row/Column + Margin/Padding
  • แก้ Hover/สีตัวอักษร + Custom CSS
  • ฝึก Responsive ให้สวยในมือถือ/แท็บเล็ต
  • เริ่มทำงานร่วมทีม + รับฟีดแบ็กจากพี่เลี้ยง
WordPress WPBakery HTML CSS UX/UI Responsive

Tools ที่ใช้

Adobe Firefly • Canva • Photoshop • WPBakery • Custom CSS

เรื่องราวการฝึกงานเดือนที่ 1

เดือนแรกคือช่วง “ปรับตัวกับของจริง” เพราะทำงานกับระบบเดิม (Legacy) ที่อัปเดตเสี่ยงต่อเว็บไซต์หลัก จึงต้องทำงานแบบแมนนวล + ระมัดระวัง และคิดเป็นขั้นตอนก่อนลงมือแก้จริง

ภาพรวมงานที่ได้รับมอบหมาย

  • ออกแบบ/ปรับปรุงหน้า About Us (thaisuggestion.com)
  • จัดโครงสร้างด้วย WPBakery (Row / Column)
  • ปรับฟอนต์ สี และองค์ประกอบตามหลัก UX/UI
  • แก้ปุ่ม Hover ให้ชัดเจนและดูเป็นมืออาชีพ
  • ใช้ HTML/CSS เสริมเพื่อแก้ข้อจำกัด Page Builder
  • ทดสอบและปรับ Responsive ให้ดูดีทุกอุปกรณ์

เหตุผลที่ต้อง “ทำงานแบบปลอดภัย”

เว็บไซต์ใช้ WPBakery เวอร์ชันเก่า ซึ่งการอัปเดตอาจทำให้ระบบผิดพลาดหรือเว็บล่มได้ จึงต้องวางแผนก่อนแก้จริงทุกครั้ง (คิดผลกระทบ / จุดที่เสี่ยง / วิธี rollback) และทดสอบซ้ำเพื่อความมั่นใจ

เรียนรู้หลัก: ทำงานกับระบบเก่าอย่างปลอดภัย, วางแผนก่อนแก้จริง

เนื้อหางานที่ทำในรอบสัปดาห์ (Week 1)

สัปดาห์แรกได้โฟกัสที่การออกแบบเว็บไซต์ thaisuggestion.com เป็นหลัก โดยเริ่มพัฒนาทักษะด้านการออกแบบมากขึ้น ทั้งการจัดวางเลย์เอาต์ การเลือกใช้สี ฟอนต์ และองค์ประกอบตามหลัก UX/UI รวมถึงใช้งาน WPBakery ร่วมกับ HTML/CSS เพื่อปรับแต่งหน้าเว็บให้ตรงตามดีไซน์ที่ต้องการ พร้อมทั้งเริ่มทำงานร่วมทีมและฝึกการสื่อสารให้ชัดเจนขึ้น

สิ่งที่ได้เรียนรู้

  • ทักษะออกแบบหน้าเว็บตามหลัก UX/UI
  • การใช้เครื่องมือกราฟิก (Firefly/Canva/Photoshop) เพื่อปรับภาพและคอนเทนต์
  • HTML/CSS เพื่อแก้โครงสร้าง เพิ่มลูกเล่น และแก้ปัญหา layout
  • การใช้ WPBakery ควบคู่กับโค้ดเพื่อให้ได้ผลลัพธ์ตามต้องการ
  • การทำงานเป็นทีม + แชร์ไอเดีย + รับ feedback

ปัญหาที่พบ

  • การทำงานร่วมกันระหว่าง HTML/CSS กับ WPBakery ต้องเข้าใจทั้งธีมและระบบ Page Builder
  • บางดีไซน์ต้องใช้ Custom CSS และต้องทดลองหลายรอบกว่าจะตรงแบบ
  • Responsive ยังต้องฝึกให้สวยทุกหน้าจอ (มือถือ/แท็บเล็ต)
  • ช่วงแรกยังไม่ชิน Row/Column ทำให้เลย์เอาต์บางส่วนไม่เป็นระเบียบ
  • เจอปัญหา Hover แล้วสีตัวอักษรหาย ต้องแก้ด้วย CSS override

ความคิดเห็นของพี่เลี้ยง

พี่เลี้ยงชื่นชมความตั้งใจในการเรียนรู้และการปรับตัวกับเครื่องมือที่แตกต่างจากที่เคยใช้ (เช่น Elementor/Astra) พร้อมแนะนำให้พัฒนาทักษะ UX/UI เพิ่ม เช่น โทนสี การจัดวาง และองค์ประกอบภาพ เพื่อให้นำเสนอผลงานได้มืออาชีพขึ้น

“ในการทำงานจริง เราไม่ได้เลือกเครื่องมือเสมอไป ความยืดหยุ่นและการเรียนรู้เร็วคือสิ่งสำคัญ”

สรุปผล

แม้จะเป็นระบบเก่าและต้องทดลองหลายรอบ แต่สุดท้ายงานเดือนแรกสำเร็จลุล่วงด้วยดี และเป็นฐานสำคัญให้การพัฒนาในเดือนต่อ ๆ ไปทำได้ไวและมั่นใจมากขึ้น
