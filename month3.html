Data Flow

ภาพรวมระบบ: End-to-End Workflow

LINE OA → Make (Webhook/Filter/Iterator) → Prefill Google Form → Google Sheets → Apps Script Trigger → Notify Admin/Patient

สิ่งที่ได้เรียนรู้ (Learning)

  • Data Flow: เห็นภาพข้อมูลไหลทั้งระบบ ตั้งแต่ user message → เก็บ userId → เก็บข้อมูล → แจ้งเตือน
  • API/Integration: เข้าใจการทำงานของ Webhook + Reply/Push Message
  • Teamwork: วางแผน workflow ร่วมกัน แบ่งหน้าที่ ทดสอบร่วมกัน

Identity & Data Integrity

วางหลักการว่า LINE userId ต้องได้จาก LINE เท่านั้น และผู้ใช้ต้องเข้าฟอร์มผ่านลิงก์ที่ส่งจาก LINE เพื่อให้ userId ถูกผูกกับข้อมูลการนัดหมายอย่างถูกต้อง

Key rule: เก็บ userId ก่อน → ค่อยให้กรอกฟอร์ม

Development Process

ขั้นตอนการพัฒนา (Dev Process)

ออกแบบ workflow end-to-end → ตั้งค่า LINE OA/Messaging API → สร้าง Make Scenario → ทำ Prefill Form → เชื่อม Sheets + Apps Script Trigger → แจ้งเตือน Admin/Patient

1) End-to-End Workflow Design

กำหนดเส้นทางข้อมูล: LINE → Form → Sheet → Trigger → Notify

2) LINE OA & Messaging API Setup

เตรียม webhook / token / โครงข้อความตอบกลับ (reply/push)

3) Make Scenario Automation

Webhook → Iterator → Filter “นัดแพทย์” → ดึง userId/replyToken → สร้าง Prefill link → ส่งกลับ LINE

4) Google Form Prefill Design

ทำฟอร์มให้รองรับ Prefill เพื่อฝัง userId ลงไปอย่างถูกต้อง

5) Google Sheets + Apps Script Trigger

เขียน onFormSubmit(e) อ่านแถวใหม่ (e.values) แล้วทำงานทันที

6) Notification (Admin / Patient)

ส่ง Flex Message ให้แอดมิน + ส่งข้อความยืนยันให้ผู้ป่วย โดยใช้ userId จากชีต

ปัญหาที่พบ

  • Make แพ็กเกจฟรีมีเครดิตจำกัด → ระบบหยุดเมื่อครบโควตา
  • โครง workflow ซับซ้อน ต้องทดสอบหลายรอบเพื่อเก็บ userId ให้ถูกต้อง
  • ช่วงแรกยังงงเรื่องการ deploy/การตั้งค่า integration บางจุด

ความคิดเห็นของพี่เลี้ยง

แนะนำให้เข้าใจภาพรวมระบบอัตโนมัติให้ชัด ทดลองหลายรูปแบบ และเลือกเครื่องมือให้เหมาะกับงบประมาณในงานจริง พร้อมตรวจงานและให้คำแนะนำละเอียดทุกขั้นตอน
